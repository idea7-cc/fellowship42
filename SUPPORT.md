# Fellowship42 support

## Community support

Use GitHub issues for reproducible bugs, documentation problems, and focused
feature proposals. Search existing issues first and include the Fellowship42
release, deployment mode, expected behavior, actual behavior, and a minimal
reproduction using synthetic data.

Do not post member, donor, contribution, export, credential, token, private key,
or production provider data. Report suspected vulnerabilities through the
private process in [SECURITY.md](SECURITY.md), not a public issue.

Community support is best effort. The project currently promises no response,
resolution, uptime, recovery, or compatibility SLA. The latest stable release
and current `main` are the useful starting points for diagnosis.

## Operator responsibility

Self-hosters and independent operators are responsible for their Cloudflare
account, Access policies, domains, secrets, backups, monitoring, incident
response, legal/privacy obligations, and tested recovery. The public project
provides portable tooling and evidence contracts; it does not operate a user's
account or certify a deployment.

Read [Deploying one instance](docs/deployment.md),
[Third-party operators](docs/third-party-operators.md), and
[Instance recovery](docs/operator-recovery.md) before production use.

## Commercial support

Fellowship42 Cloud and certified-partner services, if offered, are separate
commercial relationships governed by their own written terms and service
policies. Payment, entitlement, or certification cannot disable the public
application, remove church data, prevent local export, or override local
management revocation.
